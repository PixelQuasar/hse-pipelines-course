# План выполнения ДЗ №7: Lakehouse и форматы таблиц

## Цель
Изучить и сравнить три основных формата таблиц Lakehouse: Delta Lake, Apache Iceberg и Apache Hudi. Провести эксперименты с конкурентной записью и бенчмарки производительности.

## 1. Подготовка окружения
- [x] Создать директорию `dz7` и скопировать датасеты из `dz5`.
- [x] Создать виртуальное окружение `dz7_venv`.
- [ ] Установить зависимости: `pyspark==3.5.0`, `delta-spark==3.0.0`, `pytest`, `pytest-benchmark`.
    - *Примечание:* Используем Spark 3.5, так как поддержка Iceberg и Hudi для Spark 4.x может быть нестабильной или требовать специфичных версий.

## 2. Конфигурация Spark
- Создать `dz7/conftest.py` с фикстурами для SparkSession.
- Настроить поддержку всех трех форматов:
    - **Delta Lake**: `io.delta.sql.DeltaSparkSessionExtension`, `org.apache.spark.sql.delta.catalog.DeltaCatalog`.
    - **Iceberg**: `org.apache.iceberg.spark.SparkCatalog`.
    - **Hudi**: `org.apache.hudi.spark.sql.parser.HoodieSparkSessionExtension`.
- Скачать необходимые JAR-файлы (через `spark.jars.packages`).

## 3. Эксперименты с конкурентной записью (ACID)
- **Сценарий**: Два процесса одновременно пытаются обновить одну и ту же таблицу (или вставить данные).
- **Цель**: Проверить, как форматы обрабатывают конфликты (Optimistic Concurrency Control).
- **Тесты**:
    1. `test_concurrent_write_delta`: Запуск двух потоков записи в Delta таблицу.
    2. `test_concurrent_write_iceberg`: Запуск двух потоков записи в Iceberg таблицу.
    3. `test_concurrent_write_hudi`: Запуск двух потоков записи в Hudi таблицу.
- **Ожидаемый результат**: Один процесс должен выиграть, второй - получить ошибку или (в случае Hudi/Delta с настройками) автоматически повторить попытку.

## 4. Бенчмарки производительности
- **Метрики**:
    1. **Размер на диске**: Сравнить объем занимаемого места после начальной загрузки.
    2. **Скорость чтения**: Время выполнения `count()` и фильтрации.
    3. **Скорость записи**:
        - Начальная загрузка (100%).
        - Инкрементальная вставка (10%).
        - Upsert (Merge) (10% обновлений).
- **Скрипты**:
    - `dz7/bench_storage.py`: Замер размера.
    - `dz7/bench_read.py`: Замер чтения.
    - `dz7/bench_write.py`: Замер записи/upsert.
